# kh-l3-yoji-consistency — 文書内用字用語の表記統一

> ⚠️ ステータス: L3（LLM支援）・未実装。SDK が L3 を支持した後に実装予定。本ドキュメントは仕様の先行定義です。

- **ruleId**: `kh-l3-yoji-consistency`
- **nameJa**: 文書内用字用語の表記統一
- **ガイドライン**: 標準 校正必携 第8版 (`kousei-hikkei-8`)
- **レベル**: L3（LLM支援）
- **想定 severity**: warning
- **適用モード案**: novel / official / academic / blog

## 意図

文書内で同一の語（概念）が複数の表記で混在している場合（表記ゆれ）を検出し、統一を促す。書籍は素読み校正のチェックポイントとして、用字・仮名遣い・送り仮名の方針との整合性確認、数字表記や固有名詞の統一、約物の誤用点検などを明記している。

## 書籍根拠

標準 校正必携 第8版（日本エディタースクール）校正の作業順序とチェックポイント

- 初校・確認事項：共著・共訳書では著訳者による表記の不統一が起こりやすいため、あらかじめ対策を立てることを推奨しており、文書全体にわたる表記統一を校正着手前の確認事項として位置付けている。
- デジタル原稿の注意：データ原稿では同音・同訓異義語などの間違いが編集段階で見落とされることがあるとして、素読み校正時に十分な注意を促している。
- 素読みの個々の注意点：漢字の字体・仮名遣い・送り仮名の方針との整合性確認、数字表記・固有名詞の統一確認、言葉・文字の使い方の適否点検などが列挙されている。

## 動作原理

**LLM への入力**: 文書全体のテキストと、検出対象の語種の分類ヒント（漢字表記 / 仮名表記 / 混用の典型例リストなど）。

**LLM の判断基準**:
1. 文書内に同一と見なせる語（読みが同一 or 意味が同一）が複数の表記形式で登場していないか走査する。
   例: 「子供」と「子ども」、「行なう」と「行う」、「プログラム」と「プログラム」（全角/半角混在）。
2. 出現頻度が多い表記を「基準表記」と推定し、少数派の異表記を「ゆれ」として報告する。
3. 固有名詞（人名・地名・作品名）・引用・意図的な揺れ（方言・キャラクター語法）は除外する。
4. 統一すべき表記の候補を提示するが、自動修正はしない（文脈・著者方針による）。

**LLM の出力**: ゆれのある語グループ（表記バリアント×出現箇所リスト）、推奨統一案、疑問提起。

## なぜ L1/L2 で不可能か

- L1（正規表現）：「子供」と「子ども」を同一語と認識するには読み・意味の理解が必要。固定パターンで全語彙をカバーするのは不可能。
- L2（形態素解析）：表記バリアントが同一語かどうかは辞書引きだけでは判断できない。例えば「行なう」と「行う」は形態素解析で別見出しになり、文書スコープの統計処理（全文スキャン → 多数決）を形態素解析単体では実行できない。文書全体を俯瞰した意味的同一性の判断が L3 に必要な根拠となる。

## 例

### Positive（指摘なし）

```
子どもたちは元気に遊んでいた。……子どもの頃の思い出。
```
「子ども」で統一されている。指摘不要。

```
行事を行う。計画を行う方針だ。
```
「行う」で統一されている。指摘不要。

### Negative（指摘あり）

```
子どもたちは元気に遊んでいた。……子供の頃の思い出。
```
「子ども」（2箇所）と「子供」（1箇所）が混在。→ 警告: 「子ども／子供」の表記が文書内で混在しています。多数派の「子ども」に統一することを検討してください。

```
プロジェクトを行なう際に注意する。……計画を行う方針だ。
```
「行なう」と「行う」が混在（送り仮名の不統一）。→ 警告: 「行なう／行う」の送り仮名が混在しています。書籍方針に従い統一を検討してください。

## 偽陽性・運用注意

- 引用文・歌詞・セリフなど他者の文章を引用している箇所では表記が異なることが正当であるため、引用ブロックは除外すること。
- 文書が複数の著者・章から成る場合、各章の著者方針が異なることがある（共著・共訳書では著訳者により不統一が起こりやすい）。この場合は全文統一ではなく章単位での確認にとどめることが望ましい。
- 意図的な表記差（固有名詞商標名・キャラクターの方言など）を誤検知しやすいため、ユーザーが除外語を設定できる仕組みを実装時に考慮する。
- severity は error でなく warning が上限。統一すべきかどうかは著者・編集方針に委ねる必要がある。
